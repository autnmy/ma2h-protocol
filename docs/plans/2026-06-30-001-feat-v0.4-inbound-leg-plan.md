# Plan ‚Äî MA2H v0.4: the human‚Üíagent inbound leg

**Issue:** [#18](https://github.com/autnmy/ma2h-protocol/issues/18) (tracking) ¬∑
**SCP:** [#19](https://github.com/autnmy/ma2h-protocol/issues/19) ¬∑
**Design artifact:** [`docs/proposals/scp-v0.4-inbound-leg.md`](../proposals/scp-v0.4-inbound-leg.md)

## Goal

Add a **human‚Üíagent** messaging leg ‚Äî the inbound **directive** ‚Äî structurally symmetric to the v0.3
agent‚Üíhuman `ask`+response leg, reusing v0.3's envelope conventions, RFC 8785 JCS canonicalization, detached
signature, `jti`/`id` dedup, ¬ß9.4 SSRF controls, and ¬ß3.1 durability. **Additive and backward-compatible**
(MINOR bump 0.3 ‚Üí 0.4): no v0.3 leg changes; a 0.4 Hub stays compatible with 0.3 agent‚Üíhuman envelopes.

## Decided design (non-negotiable, per the brief)

- One inbound message type ‚Äî `directive` ‚Äî authored by a Hub-attested `human:<id>`, addressed to
  `agent:<id>` (the `notify`/instruction mirror; inbound ask/task deferred).
- Pull-first: a durable per-agent **mailbox** drained via an authenticated poll ‚Äî at-least-once + `jti`/`id`
  dedup + explicit consume/ack, FIFO per agent ‚Äî plus an optional webhook mirroring v0.3 push.
- Agent authenticates with its existing bearer token to drain; each delivery carries a Hub detached
  signature (¬ß9.7) the agent verifies (mirror v0.3 pushed-response verification).
- Discovery at `/.well-known/ma2h` (`inbound` capability object).

Wire-name decisions (rationale in the SCP): type `directive`; endpoints `GET /v1/inbox` + `POST
/v1/inbox/ack`; capability field `inbound`; SQS-style peek+ack with visibility timeout; FIFO on first
delivery, `id`-dedup for correctness; un-acked retained to a TTL, drop on expiry, no response leg; MUST-verify
the signature on both channels; **defer** human‚Üíagent ask/task.
